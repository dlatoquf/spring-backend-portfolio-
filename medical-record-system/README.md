## 📌 프로젝트 소개
환자 정보와 진료 기록을 통합 관리할 수 있는 웹 기반 의료 데이터 시스템입니다.  

단순 CRUD 구현을 넘어  
👉 **환자 정보 → 진료 기록 → 상태 관리까지의 데이터 흐름을 고려하여 설계**하였으며,  
의료 데이터의 특성을 반영하여 **데이터 간 관계와 정합성을 유지하는 구조**로 구현했습니다.

## 💥 트러블 슈팅

### 문제  
환자 정보와 진료 기록 간 관계 설정이 명확하지 않아  
데이터 조회 및 관리 시 비효율 발생  

### 해결  
- 환자 테이블과 진료 기록 테이블 간 관계 재설계  
- 참조 구조를 명확히 정의하여 데이터 연결  

### 결과  
데이터 조회 성능 및 관리 효율 개선  
데이터 정합성을 유지하는 구조 확보  

---

## 📊 데이터 흐름
1. 환자 정보 입력  
2. 서버에서 데이터 검증 및 처리  
3. DB에 환자 정보 및 진료 기록 저장  
4. 조회 요청 시 조건 기반 데이터 반환  

---

## 👨‍💻 개발 회고

이 프로젝트를 통해 단순 CRUD 구현을 넘어  
**데이터 구조와 흐름을 함께 고려하는 백엔드 개발의 중요성**을 체감했습니다.  

특히 환자 정보와 진료 기록을 연결하는 과정에서  
데이터 간 관계를 어떻게 설계하느냐에 따라  
시스템의 효율성과 유지보수성이 크게 달라진다는 점을 이해하게 되었습니다.  

또한, 사용자 입력부터 데이터 저장, 조회까지 이어지는 흐름을 직접 구현하며  
**웹 서비스의 전체 동작 구조를 실무 관점에서 경험**할 수 있었습니다.  

이 경험을 통해 단순 기능 구현을 넘어  
데이터 중심으로 시스템을 설계하는 백엔드 개발자로 성장하고자 하는 방향성을 확립했습니다.
