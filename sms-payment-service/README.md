# 💳 SMS Payment Service
> SMS 인증 기반 결제 요청 및 승인 처리 시스템 (REST API 기반)

## 📌 프로젝트 개요

* 프로젝트명: SMS Payment Service
* 개발 기간: 2025.07 (약 1개월)
* 개발 인원: 1명 (단독 프로젝트)
* 역할: API 설계, 비즈니스 로직 구현, DB 설계 및 데이터 처리
* 플랫폼: Web (REST API)

---

## 🛠 개발 환경

* Language: Java 17
* Framework: Spring Framework
* DB: MariaDB
* API: RESTful API
---

## 🚀 핵심 성과
- REST API 기반 결제 처리 시스템 구현
- 결제 요청 → 인증 → 승인 흐름 설계
- 트랜잭션 처리로 데이터 정합성 확보
- 오류 발생 시 롤백 처리 구현

---

## 📌 프로젝트 소개
SMS 인증을 기반으로 결제 요청 및 승인 처리를 수행하는 백엔드 시스템입니다.  

사용자의 결제 요청부터 인증, 승인까지의 흐름을 REST API로 설계하여  
안정적인 결제 처리 시스템을 구현했습니다.

---

## 🛠 기술 스택
- **Backend**: Java, Spring Framework  
- **Database**: MariaDB  
- **Tools**: Git  

---

## ⚙️ 주요 기능
- SMS 결제 요청 처리
- 인증 코드 검증
- 결제 승인 및 취소
- 결제 상태 관리

---

## 🔥 핵심 구현

### 1. 결제 API 흐름 설계
- 결제 요청 → 인증 → 승인 프로세스 구성
- 상태값 기반 결제 흐름 관리

---

### 2. 트랜잭션 기반 데이터 처리
- 결제 도중 오류 발생 시 롤백 처리
- 데이터 정합성 유지

---

### 3. REST API 구현
- 결제 요청 및 승인 API 구현
- JSON 기반 데이터 처리

---

## 💥 트러블 슈팅

### 문제
결제 처리 도중 오류 발생 시 데이터 불일치 문제 발생

### 해결
트랜잭션 처리 및 상태값 기반 로직 추가

### 결과
결제 안정성 향상 및 데이터 일관성 확보

---

## 📊 데이터 흐름
1. 사용자 결제 요청
2. SMS 인증 요청
3. 인증 완료 후 결제 승인
4. DB에 결제 상태 저장
