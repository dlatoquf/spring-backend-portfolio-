## 📌 프로젝트 소개
SMS 인증을 기반으로 결제 요청 및 승인 처리를 수행하는 백엔드 시스템입니다.  

단순 API 구현을 넘어  
👉 **결제 요청 → 인증 → 승인까지의 전체 흐름을 상태 기반으로 설계**하여  
안정적인 결제 처리 시스템을 구현했습니다.  

특히 결제 과정에서 발생할 수 있는 오류 상황을 고려하여  
트랜잭션 처리 및 상태값 기반 로직을 적용함으로써  
데이터 정합성을 유지하도록 설계했습니다.

---

## ⚙️ 주요 기능
- SMS 결제 요청 처리  
- 인증 코드 검증  
- 결제 승인 및 취소 처리  
- 결제 상태 관리  

---

## 🔥 핵심 구현

### 1. 결제 요청 및 SMS 발송 처리 (결제 흐름 설계 포함)

- 고객에게 SMS 링크를 발송하여 결제 요청을 진행하고,  
  승인·취소 및 상태 조회가 가능하도록 전체 결제 프로세스를 설계  
- 화면에서 입력한 이름, 휴대폰번호, 금액, 주문번호, 기한 등을 기반으로  
  REST API를 호출하여 결제 요청 처리  
- 주요 처리 흐름  
  UI 입력 → POST /sms/payment → DB 저장 (TBTR_SMS 등) → SMS 게이트웨이 전송  
- 결제 요청 → 인증 → 승인으로 이어지는 흐름을  
  상태값 기반으로 관리하도록 설계  
- 데이터 품질 보장  
  필수값 및 형식 검증을 통해 잘못된 데이터의 DB 반영을 사전 차단  
- 안정성 확보  
  요청/응답 로깅 및 예외 처리로 오류 원인 추적 가능  
- SMS 링크는 결제 요청 전용으로 사용되도록 설계

---

### 2. 트랜잭션 기반 데이터 처리
- 결제 도중 오류 발생 시 전체 작업 롤백 처리  
- 데이터 일관성을 유지하기 위한 트랜잭션 적용  
- 실패 케이스를 고려한 예외 처리 로직 구현  

---

### 3. REST API 구현
- 결제 요청 / 승인 / 취소 API 설계 및 구현  
- JSON 기반 데이터 송수신 처리  
- 클라이언트와 서버 간 명확한 역할 분리

## 💥 트러블 슈팅

### 문제
결제 처리 도중 오류 발생 시  
일부 데이터만 반영되는 데이터 불일치 문제 발생  

### 해결
- 트랜잭션 처리 적용으로 전체 작업 단위 관리  
- 상태값 기반 로직을 도입하여 처리 흐름 제어  

### 결과
데이터 정합성을 유지하면서  
안정적인 결제 처리 흐름을 확보  

---

## 📊 데이터 흐름
1. 사용자 결제 요청  
2. SMS 인증 요청 및 코드 발송  
3. 인증 완료 후 결제 승인 처리  
4. DB에 결제 상태 저장

## 👨‍💻 개발 회고
이 프로젝트를 통해 REST API 기반 서비스 설계 경험을 쌓을 수 있었으며,  
단순 기능 구현을 넘어 **결제 흐름 전체를 설계하는 경험**을 할 수 있었습니다.  

특히 결제와 같이 데이터 정합성이 중요한 도메인에서는  
단순 로직 구현이 아닌  
**트랜잭션 처리와 상태 관리가 핵심이라는 점**을 체감했습니다.  

결제 요청 → 인증 → 승인으로 이어지는 과정에서  
상태값을 기준으로 흐름을 제어하면서  
**안정적인 서비스 구조 설계의 중요성**을 이해할 수 있었습니다.  

또한, 오류 발생 시 롤백 처리와 예외 로직을 구현하면서  
실제 운영 환경을 고려한 방어적인 코딩의 필요성도 경험했습니다.  

앞으로는 Spring Boot 및 JPA를 활용하여  
보다 **확장성과 유지보수성을 고려한 API 설계 역량**을 발전시킬 계획입니다.
